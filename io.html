Multimodal World I/O

If VWM is the authoritative model of reality, then almost anything that can be measured or actuated can eventually become an input or output — multimodal world I/O around one voxel kernel.

Vision catalog, not a shipped-driver checklist

This page lists the intended I/O surface for an authoritative world model. Many modalities already plug into capture / query / sync / trust; others are horizon adapters on the same contracts. Reality becomes evidence; actuation becomes validated operations — never a second source of truth.

Reality (sensors · humans · machines · archives) │ ▼ Observation / Event intake → Authoritative Voxel World │ ├── world.query (read) ├── world.sync (replicate) ├── world.trust (authorize ops) └── actuation / views / APIs → Reality

Internal Inputs

These are not sensors — they become observations inside VWM, still under provenance and confidence rules.

AI object detections Semantic segmentation SLAM poses Human annotations Robot observations Simulation predictions Planning results External APIs Historical archives Knowledge graph imports

Predictions stay labeled as predictions. Annotations and detections carry evidence — they never silently become geometry authority.
