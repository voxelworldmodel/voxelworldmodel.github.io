Invariant across every future milestone

The Adaptive Voxel World remains the single source of geometric truth. Perception proposes observations; the Evidence Engine decides belief; Components preserve identity; the SVO accelerates queries; the runtime coordinates incremental recompiles; every visualization is a derived view. Later milestones add systems — they do not replace the world.

The major subsystem stack is shipped through V16 (world → compiler → visualization). V13 production Gaussians and V15 Part 2 (browser migration) remain open polish on that stack; V17+ are capability layers. Items marked Foundation laid already have scaffolding and become first-class products in the listed milestone.

V13 — Production Gaussian Engine

Turn the V12/V15/V16 Gaussian path into a production observation pipeline: streaming rasterization plus optional local and global frame enhancers — without ever letting AI become the world authority.

Adaptive Voxel World │ ▼ ══════════════════════════ World Compiler ══════════════════════════ │ ▼ Gaussian Scene Artifact │ ▼ ══════════════════════════ Gaussian Rasterizer ══════════════════════════ │ ▼ Gaussian Frame + Metadata │ ┌────┴────┐ ▼ │ Local AI │ (materials, textures, fine detail) │ │ ▼ │ Enhanced │ └────┬────┘ ▼ Final Display Frame │ ▼ Global AI (optional) (HDR, grading, atmosphere, bloom, DOF, temporal, style) │ ▼ Monitor

Goals

  • LOD & frustum streaming — serve only visible / near chunks; screen-space error drives splat density.
  • VRAM / resource manager — adaptive budgets for splat counts, atlas sizes, and compile queues.
  • Progressive initial compile — background progressive chunk build + live partial views.
  • Gaussian rasterizer as a V16 renderer — consumes only gaussian_scene artifacts; produces frame + G-buffers (RGB, depth, normals, motion, semantics, materials).
  • Layer 1 — Local Frame Enhancer — AI conditioned on per-object / per-voxel metadata (semantic, material, moisture, season, confidence…); improves surface detail, reflections, edges, AA — never invents occupancy.
  • Layer 2 — Global Frame Enhancer — AI conditioned on scene metadata (weather, fog, sun elevation, exposure, time of day…); HDR, grading, atmosphere, bloom, DOF, temporal, upscaling, stylization.
  • Independent toggles — local_ai / global_ai on|off: pure Gaussian, photoreal surfaces only, fast cinematic, or maximum realism.
  • Optional WebGPU path — GPU splat sort / raster when available; WebGL elliptical quads remain the fallback.

Definition of done

  • Viewer streams Gaussians under a declared VRAM budget without dropping the authoritative world.
  • Local and global enhancers are optional plugins over the same raster output; disabling both yields pure deterministic Gaussian display.
  • Enhancers never write voxels, Components, or evidence — observation only.
  • Benchmarks separate raster cost from local AI cost from global AI cost.

Depends on

V12 Gaussian scene · V15 compiler artifacts · V16 visualization engine · homepage / paper enhancement path. See also Frame Enhancement on the homepage.

V14 — Complete Registration & Mapping ✓

Delivered. Maintained SE(3) pose graph with Gauss-Newton optimization, loop closure, landmarks, anchors, uncertainty, multi-capture fusion, and incremental voxel corrections through the V11 runtime. Measured: 12-node loop drift 8.99 m → 0.0 m in 70 ms. Full report: updates.html#v14.

Part 1 delivered. Single World Compiler Runtime owns all derived-artifact compilation; five built-in plugins wrap real Gaussian / SVO / mesh / semantic / lighting logic; incremental, transactional, cached, deterministic-where-applicable, and extensible without touching the runtime class. Next Planned

V17 — World Simulation

Run deterministic physics, agents, and processes on the authoritative voxel world and its Components — simulation as a first-class citizen, not a side demo.

Goals

  • WorldClock + EventScheduler — discrete, reproducible ticks; recorded timelines for replay.
  • Physics on voxels & Components — collision from SVO / hull compilers; gravity, constraints, simple fluids/airflow as optional solvers.
  • Agent & automation runtime — scripted and AI agents act through the same APIs as robots and players.
  • Dirty-aware sim — world edits invalidate only affected sim islands; no full-world reset.
  • Replay & what-if branches — fork a revision, simulate, compare, optionally commit.

Definition of done

  • Same initial revision + same inputs → identical sim trajectories (determinism test).
  • Walk / Fly collision in the viewer can optionally consume the production physics hull compiler.

Depends on

V15 collision / navmesh compilers · V10 Components · V11 events & revisions · SVO collision suite.

Next Planned

V18 — Digital Twin Platform

Keep real devices and spaces continuously synchronized into the same world: sensors propose observations, the twin stays editable and queryable, and actuation goes back out through explicit channels.

Goals

  • Live sensor adapters — RGB-D, LiDAR, IMU, IoT, industrial buses → Captures / observations with provenance.
  • Bidirectional twins — Component state mirrors device state; commanded state changes emit actuation events (never silent).
  • Multi-site federation — regions / buildings as hierarchical Components with sync policies.
  • SLA & health — sensor coverage, staleness, confidence decay (Evidence Engine aging) as first-class twin health metrics.
  • Security & tenancy — who may observe, edit, or actuate which Components.

Definition of done

  • A live device stream updates evidence and Components without stopping the viewer or sim.
  • trace(point) still reaches the originating Capture / device message.

Depends on

V10 upload / Captures · V11 evidence aging · V14 registration for relocating sensors · V17 for actuated sim feedback.

Next Planned Foundation: V10 query + V11 evidence

V19 — Spatial Database & Query Engine

Scale the world into a persistent spatial database: disk-backed evidence and revisions, indexed queries, and analytics that stay honest at city / campus scale.

Goals

  • Disk persistence — evidence, revisions, Captures, Components, compiler artifacts survive restart (closes V11 in-memory limit).
  • Spatial & semantic indexes — SVO + secondary indexes for materials, types, confidence, time, provider, capture.
  • Query language v2 — joins across Components / Captures / evidence / trajectories; streaming query results.
  • Vectorized evidence — replace per-voxel Python dict hot paths for large worlds (V11 honest note).
  • Analytics views — coverage, uncertainty maps, change detection between revisions as compiled analytics.

Definition of done

  • Cold start restores an identical revision with checksummed artifacts.
  • Declared query SLAs on hallway-scale and a larger synthetic campus fixture.

Depends on

V10 query API · V11 evidence / revisions · V15 artifact store · SVO as spatial accelerator.

Next Planned

V20 — Plugin SDK & World Operating System

Productize the World OS: a stable SDK so third parties ship providers, compilers, renderers, sensors, and tools without forking the core — the operating system for spatial worlds.

Goals

  • Public SDK — typed APIs for providers, compilers, renderers, query extensions, and actuation; versioned compatibility.
  • Plugin lifecycle — install, sandbox permissions, resource budgets, hot-reload where safe.
  • World OS shell — graduate the viewer panels (upload, search, runtime, reconstruction) into a coherent operator UI with history / revision browsers.
  • Packaging & distribution — worlds as portable bundles (voxels + evidence + components + selected compiled views).
  • Compliance hooks — audit logs for edits, uploads, and actuations.

Definition of done

  • An external sample plugin (provider or renderer) builds against the SDK and passes CI without core patches.
  • World bundle export / import round-trips with provenance intact.

Depends on

V15–V16 plugin surfaces · V10–V12 server APIs · V19 persistence for durable worlds.

Horizon Horizon

V21+ — World Intelligence & Autonomous Planning

Agents that understand, plan, and act inside the same authoritative world — using evidence, semantics, and simulation — without treating a neural render as reality.

Themes (multi-milestone)

  • World understanding — long-horizon semantic graphs, affordances, and task-relevant abstractions over Components.
  • Autonomous planning — planners that query the spatial DB, simulate branches (V17), and propose edits or actuations with human-visible justification.
  • Attention & uncertainty-driven sensing — next-best-view / next-best-sensor policies that maximize evidence gain (ties to Evidence Engine confidence).
  • Multi-agent coordination — robots, AIs, and humans sharing one revision timeline with conflict policies.
  • Prediction layers — forecast views (occupancy, traffic, energy) as compiled analytics — always labeled predicted, never smuggled into authority.
  • Continuous learning loops — experience from twins and sims feeds appearance / perception training under frozen metrics (same honesty rules as appearance v2.x).

Non-goals (still)

  • The renderer never becomes the simulator.
  • Generated imagery never silently becomes occupancy or Components.
  • Autonomy never skips provenance — every act traces to world state and policy.

Depends on

V16–V20 as the operable platform · V11 evidence · V17 sim · V18 twins · V19 query scale.
